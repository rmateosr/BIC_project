# BIC_project — Time-based ASM detection

Detect **changes in allele-specific methylation (ASM) between two (or more)
time points** from long-read sequencing data (Oxford Nanopore or PacBio HiFi),
using a four-way **BIC model competition** that distinguishes static ASM from
ASM that *emerges over time*, and likewise distinguishes a single shared
methylation trajectory from a haplotype-split one.

## The four competing models

Each genomic window (10 CpG sites by default) is scored under four models and
the lowest-BIC winner is reported.

| Conversational name           | Meaning                                                                 | TSV winner label  |
|-------------------------------|-------------------------------------------------------------------------|-------------------|
| Shared                        | One methylation level, no haplotype split, no time change               | `shared`          |
| ASM                           | Two static haplotype methylation levels                                 | `asm`             |
| ASM with time emergence       | Two haplotype levels **plus** a free per-time-point altered fraction    | `asm_time`        |
| Shared with time emergence    | One shared methylation level **plus** a free per-time-point altered fraction | `shared_time` |

The time-emergent models leave the per-time-point altered fraction
`π[h, 1, t]` free for every `t ≥ 2` with `π[·, 1, 1] = 0`; they do **not**
assume a linear 0 → π_final ramp.

Math derivation: see [`em/`](em/) and the docstrings in
`em/EMfunctions_timebased_diploid_aware_model.py` and
`em/EMfunctions_timebased_shared_time_emergent_model.py`.

## Repository layout

```
.
├── em/                       EM + BIC core (4-way driver, tests, synthetic generators)
├── upstream/                 Shared per-chrom/per-hap pipeline scripts (bamsplit,
│                             modkit, filter, coordfix, divide-into-regions)
├── timebased_from_tsv/       Entry point for nanopolish-era methylation TSVs
├── timebased_from_bam/       Entry point for modBAMs with MM/ML tags
└── submit_downstream_after_m2r.sh
                              Coordinator job that chains the downstream stages
                              once per-time-point read tables are ready
```

`timebased_from_tsv/` and `timebased_from_bam/` are thin wrappers — most files
inside `scripts/` are symlinks back into `upstream/scripts/`. The EM driver
referenced by both is `em/BIC_algorithm_timebased_4way.py`.

## Quick start

Pick the entry point that matches your methylation calls:

| Input type                                                  | Use                                             |
|-------------------------------------------------------------|-------------------------------------------------|
| Nanopolish TSV (`.tsv.gz` + `.tbi`) + phased BAM            | [`timebased_from_tsv/`](timebased_from_tsv/README.md) |
| Phased modBAM with MM/ML tags (Dorado / PacBio HiFi Revio)  | [`timebased_from_bam/`](timebased_from_bam/README.md) |

Both expose a single `run_pipeline.sh` that chains the full stack and runs
the 4-way BIC EM at the end.

For a self-contained synthetic example (no real data needed), see
[`em/example_dataset/`](em/example_dataset/README.md) — generated by
`em/synthetic/generate_synthetic.py`, with a `manifest.tsv` answer key.

## Requirements

- **Python ≥ 3.10** — `numpy`, `pandas`, `scipy`, `pytest`
- **R ≥ 4.0** — `dplyr`, `stringr`, `data.table`
- **samtools ≥ 1.19**
- **bgzip + tabix** (for indexed nanopolish TSVs)
- **modkit** (BAM path only) — download from
  https://github.com/nanoporetech/modkit/releases and place the binary at
  `upstream/bin/modkit` (gitignored; a relative symlink at
  `timebased_from_bam/bin/modkit` resolves to it)
- **Reference genome** matching your BAMs (e.g. GRCh38)
- A job scheduler — the pipeline scripts assume **SGE** (`qsub`), but the
  per-step shell scripts can be adapted to SLURM with minimal changes

## Tests

```bash
cd em
pytest tests/
```

Covers: EM convergence on synthetic data, BIC parameter counting for all four
models, M-step closed-form updates, BIC `n` correction, reproducibility under
fixed seeds, synthetic-truth round-trips for the 3-way and 4-way drivers.

## Inputs and outputs

Per-time-point inputs:

| What                                  | Format                                                            |
|---------------------------------------|-------------------------------------------------------------------|
| Phased BAM                            | Phased by WhatsHap (HP tag) — required by both entry points       |
| Methylation calls                     | Nanopolish TSV (TSV entry) **or** MM/ML tags in BAM (BAM entry)   |
| Reference genome                      | FASTA matching the BAM coordinates                                |

Per-chromosome output `<chr>_timebased_ASM.tsv`:

| Column group       | Columns                                                                                          |
|--------------------|--------------------------------------------------------------------------------------------------|
| Coordinates        | `chrom`, `start`, `end`, `n_cpgs`                                                                |
| Per-model BIC      | `BICsinglecomp` (shared), `BICmiddlecomp` (asm), `BICpaircomp` (asm-time), `BIC_shared_time`     |
| Winners            | `BIC_3way_winner` (legacy 3-way), `BIC_4way_winner` (string: `shared` / `asm` / `asm_time` / `shared_time`) |
| Fits               | `pi_altered_t*_*` (per-time-point altered fractions), `theta_*` (per-allele baseline rates)       |
| Diagnostics        | `em_iterations_*`, `total_reads`, log-likelihoods                                                |
